# orders update

## 功能定位

這是一個交易控制命令：**改單 (Update Order)**。
當你有一筆已經在交易所排隊的掛單（例如限價買進 10 張台積電），但你想要更改其**委託數量 (減單)** 或 **價格 (改價)** 時，可以使用此命令，省去你手動「先刪再下」的步驟。

## 適用情境

- 掛單遲遲未成交，想提高委託買價或降低委託賣價（追價）。
- 取消部分委託數量。例如下了 10 張，目前看壞行情，把剩下的未成交量改成只保留 2 張。

## 命令語法

```powershell
python scripts/shioaji_cli.py orders update --order-id <委託單編號> [--price <新價格>] [--quantity <新數量>]
```

## 參數詳解

| 參數 | 必填 | 說明 |
|---|---|---|
| `--order-id` | **是** | 目標委託的 `id`（本地 ID）或是券商端給予的 `seqno` (委託書號)。建議先用查詢清單確認。 |
| `--price`    | 否   | 要更改的**新價格**。如果未帶此參數，則保留原掛單價格。 |
| `--quantity` | 否   | 要更改的**總委託數量。 注意，這是改為這個數量，而不是減少這個數量！** |

*(備註：`--price` 與 `--quantity` 至少必須提供其中一個，否則命令會檢查報錯。)*

---

## 執行邏輯與狀態變更

在 Shioaji 的底層，`api.update_order(trade)` 封裝了不同交易所對於改單的複雜邏輯：
1. 先利用 `--order-id` 從本地快取找出該筆 `Trade` 物件。
2. 呼叫 `update_order`。
   - **針對台股 (證券)**: 台灣證券交易所其實硬性規定**不能直接改價**，只能「減量」(減單)。如果你對台股下了改價指令，API 底層實作其實是**幫你送出刪除舊單，並且重新送出一張新價格的新委託單**。
   - **針對期貨**: 期貨交易所支援改價也支援減量。

## 實戰與進階範例

### 1. 減單 (減少委託數量)
原本下單 10 張只成交了 2 張，你想把剩餘的 8 張掛單縮減成 3 張（也就是還要量 = 3）。
```powershell
python scripts/shioaji_cli.py orders update --order-id 45d3e090 --quantity 3
```

### 2. 追價 (修改委託價格)
原本掛 750 元想買，一直沒買到，現在想抬高價格到 755 元再排一次。
*(若是台股，這樣做會導致原始訂單被 Cancel，並發出一張新的 Pending單)*
```powershell
python scripts/shioaji_cli.py orders update --order-id 45d3e090 --price 755
```

## 注意事項與雷區

1. **不可「加量」**: 幾乎所有的交易所機制都 **不允許增加委託數量**。如果你把 `--quantity` 設得比目前送出且未成交的量還要大，這筆 update 請求會被主機駁回。若要買更多，請另外發起一次 `orders place`。
2. **改價將喪失排隊優先權**: 在「價格優先、時間優先」的撮合機制下，如果你對一筆單子進行「改價」，就算只是微調，這張單的戳記時間都會被重置，你必須重新去新的價格層隊伍最後面排隊。
3. **沒有拿到 id 前不能改**: 若是透過極速非阻塞下單 (`timeout=0`) 送出的單，如果在主機還沒回傳 `seqno` 或完成 `PendingSubmit` 確立之前就急著 update，可能會因為無單可改而報錯。建議先 `update-status` 再進行改單。
