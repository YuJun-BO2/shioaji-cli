# accounts pnl

## 功能定位

這是一個帳務歷史查詢命令：**查詢已實現損益 (Realized Profit & Loss)**。
與 `positions` 查詢「未平倉」的帳面浮動損益不同，`pnl` 專門用來計算你已經「完成買賣結清」的交易實質獲利或虧損，是非常重要的對帳與績效回溯工具。

## 適用情境

- 週末檢視自己過去一個月來的投資績效與勝率。
- 查詢某天特定標的的出場總額與最終賺賠 (`pnl`) 及報酬率 (`pr_ratio`)。
- 取得已結清部位的流水號 (`id`) 以便進一步抓取細部的手續費與稅金 (`pnl-detail`)。

## 命令語法

```powershell
python scripts/shioaji_cli.py accounts pnl --start <開始日期> --end <結束日期> [--account-kind <帳別>]
```

## 參數詳解

| 參數 | 必填 | 預設值 | 說明 |
|---|---|---|---|
| `--start` | **是** | 無 | 查詢區間起始日，格式 `YYYY-MM-DD`。 |
| `--end` | **是** | 無 | 查詢區間結束日，格式 `YYYY-MM-DD`。 |
| `--account-kind` | 否 | `any` | 指定要查的帳號類別 (`stock` 證券 或 `futures` 期權)。 |
| `--account-id` | 否 | 無 | 若有同名憑證但不同身分證或分支機構的帳戶，請輸入指定 ID。 |

---

## 核心回傳資料結構 (`ProfitLoss` 物件清單)

依據你查詢的是股票或期貨，回傳的屬性稍有不同，但皆以陣列形式呈現：

### 共通與重要欄位

| 欄位名稱 | 說明與實務應用 |
|:-------|:-----------------------------------------------------------|
| `id`   | 此筆已平倉紀錄的內部系統識別碼。如果你需要查這筆單被收了多少手續費，必須拿它去下 `accounts pnl-detail`。 |
| `code` | 標的代碼（如 `"2330"`）。 |
| `quantity` | 這個平倉明細的成交數量。 |
| **`pnl`** | **已實現損益絕對值** (新台幣)。已扣除交易稅與手續費後的淨利或淨損，正數為賺，負數為賠。 |
| `date` | 平倉結算日。 |

### 證券專屬重點欄位
| 欄位名稱 | 說明 |
|:-------|:-----------------------------------------------------------|
| `price`| 建倉(買/賣出)的平均成本價。 |
| `pr_ratio`| **報酬率 (Profit Ratio)**。例如 `0.123` 代表獲利 12.3%。 |
| `cond` | 交易條件（現股/融資/融券等）。 |

### 期貨專屬重點欄位
| 欄位名稱 | 說明 |
|:-------|:-----------------------------------------------------------|
| `entry_price` | 建倉成本價格。 |
| `cover_price` | 平倉出場價格。 |

## 實戰範例

### 1. 查詢台股上個月的獲利
```powershell
python scripts/shioaji_cli.py accounts pnl --account-kind stock --start 2024-03-01 --end 2024-03-31
```

### 2. 查詢台指期當沖紀錄的已實現損益
```powershell
python scripts/shioaji_cli.py accounts pnl --account-kind futures --start 2024-04-01 --end 2024-04-01
```

## 注意事項與雷區

1. **區間限制**: 底層 API 有限制單次查詢的時間跨度與資料筆數。如果查詢過長（例如一次查 3 年），可能會觸發 Timeout，建議分批按月查詢。
2. **沒有手續費明細**: 這個陣列雖然有算好手續費後的 `pnl`，但如果你需要向會計師報帳或檢視實收稅額，這張表裡沒有單獨的稅金欄位。你必須拿這裡抽出來的 `id` 去執行 `accounts pnl-detail` 或者是直接去看 `accounts settlements` (交割試算)。
3. **資料延遲問題**: 如果你是「今天」才剛賣掉的股票，可能要等當天盤後結帳時間過後，才能夠在此清單中查到精確的已實現損益，盤中可能只會查到昨天的已平倉單。
