# accounts balance

## 功能定位

查詢**證券帳戶**目前的銀行交割帳戶可動用餘額 (Account Balance)。
這是用來確認你銀行戶頭裡面到底還有多少台幣餘額可以進行台股交割扣款的基礎命令。

## 適用情境

- 證券現股買進之前，確認交割帳戶的餘額是否充足，避免違約交割。
- 收盤後或程式自動化交易前，監控剩餘可動用資金。

*(注意：此命令僅針對「證券」交割帳戶餘額；如果是「期貨」部位保證金，請使用 `accounts margin`)*

## 命令語法

```powershell
python scripts/shioaji_cli.py accounts balance
```

## 參數詳解

| 參數 | 必填 | 預設值 | 說明 |
|---|---|---|---|
| `--account-kind` | 否 | `stock` | 鎖定查詢的帳號類別。因為是查帳務餘額，預設自動鎖定為證券 (`stock`)。 |
| `--account-id` | 否 | 自動套用 | 如果你有多個證券帳戶，可輸入身分證下該帳號的 id（通常為 7 碼數字）來確切指定。未填則預設使用第一個匹配的證券帳戶。 |

---

## 核心回傳資料結構 (`AccountBalance` 物件)

執行完畢後會回傳 `AccountBalance` 的 JSON 格式。依據官方 Reference，重要欄位對照如下：

| 欄位名稱        | 原生型別          | 說明與注意事項                                                                |
|:--------------|:-----------------|:-------------------------------------------------------------------------|
| `status`      | `FetchStatus`    | API 資料抓取狀態，通常為 `Fetched` 表示成功抓取至伺服器最新餘額。                               |
| **`acc_balance`** | `float`      | **交割帳戶中的實際可動用餘額 (新台幣)**。買股前最關鍵的數字。                                    |
| `date`        | `str`            | 資料最後更新時間或資料基準時間，格式範例為 `2023-01-06 13:30:00.000000`。              |
| `errmsg`      | `str`            | 若有查詢異常或錯誤訊息會顯示於此。若為空字串 `""` 代表無錯誤。                             |

## 實戰與進階範例

### 1. 快速查詢預設證券帳戶餘額
如果這組憑證登入後只有單一證券帳號，你可以直接呼叫而不用帶任何參數：
```powershell
python scripts/shioaji_cli.py accounts balance
```

### 2. 指定特定的證券帳戶查詢
當你擁有兩個證券帳號（例如一個正常帳戶、一個財富管理帳戶），你可以指定 7 碼 ID：
```powershell
python scripts/shioaji_cli.py accounts balance --account-id 0062402
```

## 注意事項與雷區

1. **僅限證券**: 呼叫 `accounts balance` 時底層對應 `api.account_balance`，如果這時 `--account-kind` 被強制指定為 `futures`，API 可能會回傳異常或沒有意義的值，因為期貨帳戶看的是權益數與保證金 (margin)，請小心區隔。
2. **交割機制延遲**: `acc_balance` 的數據反映銀行端目前餘額 + T+2 交割影響。請仔細比對此餘額與 `accounts settlements` (交割試算) 的款項差異。
