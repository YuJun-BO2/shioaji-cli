# pnl-detail

## 功能定位

進階查詢已實現損益（Profit & Loss）的底層交易明細。這是一個第二層查詢命令，主要用於釐清單一「損益報告」為何會產生目前的利潤或虧損金額。

## 適用情境

- **成本與稅費拆解**：查證一筆平倉單的扣稅金、手續費等摩擦成本確切數字。
- **沖銷對帳**：確認系統抓取的進場單與出場單確切搭配的日期與價格，核對沖銷邏輯是否與策略預期相符。

## 命令語法

```powershell
python shioaji_cli.py accounts pnl-detail [參數選項]
```

## 參數詳解

| 參數 | 必填 | 預設值 | 可選值 | 說明 |
|---|---|---|---|---|
| `--detail-id` | 是 | 無 | `pnl`查得的 ID | 來自上一層「已實現損益」清單返回的專屬憑證 ID。 |
| `--account-kind` | 否 | `any` | `auto / any / stock / futures` | 限縮特定市場。 |
| `--account-id` | 否 | 當前預設帳戶 | 有效的帳號字串 | 限縮特定帳號。 |

## 核心回傳資料結構

執行完畢後會回傳一組 `ProfitLossDetail` 列表。每一筆代表一筆對應的建/平倉操作，常見欄位涵蓋：

- **date**: 單筆交易成交日期。
- **code**: 商品合約代碼。
- **quantity**: 成交的口數/股數。
- **price** / **cost**: 成交價格及計算後對應的總成本。
- **fee**: 該次委託所產生的券商手續費。
- **tax**: 該次交易衍生的證交稅或期交稅。

## 實戰範例

**範例 1：檢視單證券單據成本拆解**

1. 透過主查詢 `pnl` 命令查出你剛沖銷的股票有一筆損益，附帶 `id` 譬如為 2。
2. 鑽取這筆操作的「手續費、稅金與配對進口價」：
   ```powershell
   python shioaji_cli.py accounts pnl-detail --account-kind stock --detail-id 2
   ```
*說明：可精確確認多批買進而後一次賣出時的先進先出配對過程與確切成本。*

## 注意事項與雷區

1. **強依賴前置作業**：
   此命令必須依附於 `pnl` 的查詢結果。如果沒有先獲取對應的 `--detail-id` 將無法獨立執行。
2. **區別 `position-detail`**：
   `position-detail` 針對的是**未平倉**（還在帳上飄的）；`pnl-detail` 針對的則是**已平倉**（已經落袋為安或認賠殺出的）。
