# accounts list

## 功能定位

列出當前登入使用者所擁有的「所有帳號清單」。能快速檢視使用者具有權限交易的證券、期權或是複委託等帳戶列表。

## 適用情境

- 初次建置系統時，確認憑證及登入可以綁定的實際帳號數量。
- 確認 `--account-id` 需要填寫什麼數字做為輸入參數以分辨不同戶頭。

## 命令語法

```powershell
python scripts/shioaji_cli.py accounts list
```

## 參數詳解

| 參數 | 必填 | 預設值 | 可選值 | 說明 |
|---|---|---|---|---|
| 無 | | | | 本命令不需額外參數 |

## 核心回傳資料結構

回傳為登入者擁有的各帳號物件陣列，每個物件重要資訊如下：

- `person_id`: 歸屬身分證字號。
- `broker_id`: 券商代碼 (如 9A95)。
- `account_id`: 實體的交易帳號編號 (通常 7 碼數字)。
- `account_type`: 帳戶類別 (`S` 證券, `F` 期貨, `H` 海外複委託)。
- `signed`: 本帳戶具備當地憑證簽章許可狀態，為 `True` 才能下單。

## 實戰範例

```powershell
python scripts/shioaji_cli.py accounts list
```

## 注意事項與雷區

- 確認 `signed` 狀態：看到帳戶在清單中不代表你可以馬上對它下單，必須確定該帳號 `signed` 為 `True`，否則委託會被拒絕。
- API 的其他下單或查帳命令，當未帶入特定 ID 時，預設都是抓取此清單的第一筆相同屬性帳戶。
