# accounts settlements

## 功能定位

這是一個台股專屬帳務命令：**查詢交割款試算 (Settlements)**。
在台灣證券市場實施 T+2 交割制度下（今天買的股票，後天才會從銀行帳戶扣款），此命令用來幫你結算、預判接下來幾天你的銀行帳戶「總共會被扣多少錢」或「總共會匯入多少錢」。

## 適用情境

- 盤中或盤後，確認今天所有的買賣操作，在「T+2 日」那天需不需要補錢進交割銀行（避免違約交割！）。
- 自動化交易程式的可用資金防呆，例如當 T+2 交割款算出來是負數（需扣錢）且大於目前 `balance` 時，自動發送 Line 警示。

## 命令語法

```powershell
python scripts/shioaji_cli.py accounts settlements
```

## 參數詳解

| 參數 | 必填 | 預設值 | 說明 |
|---|---|---|---|
| `--account-kind` | 否 | `stock` | 必須是 `stock`。交割款是針對台股證券帳戶的制度。期貨是當日洗價，無此概念。 |
| `--account-id` | 否 | 無 | 若有多個證券帳戶，可輸入指定 ID。未填使用預設。 |

---

## 核心回傳資料結構 (`SettlementV1` 陣列)

執行後會回傳一個陣列（通常包含 3 筆資料，對應 T, T+1, T+2）。

| 欄位名稱 | 型別    | 說明與實務應用 |
|:-------|:-------|:-----------|
| `T`    | `int`  | 距離今日交割的天數。`0` 代表今天 (前天買賣今天扣款)；`1` 代表明天 (昨天買賣)；`2` 代表後天 (今天剛買賣的額度)。 |
| `date` | `str`  | 表定的銀行實際交割日期。例如 `"2024-03-20"`。 |
| **`amount`**| `float`| **當天會發生的淨收付金額 (新台幣)**。非常重要！<br/>- **大於 0 (正數)**：代表你那天是「賣大於買」，錢會撥入你的銀行。<br/>- **小於 0 (負數)**：代表那天會從你的銀行「實施扣款」。請確保銀行餘額 (`balance`) 大於此絕對值！ |

## 實戰與進階範例

### 1. 查詢未來三日交割收付狀況
```powershell
python scripts/shioaji_cli.py accounts settlements --account-kind stock
```

## 注意事項與雷區

1. **查詢時段限制**: 根據 API 設計與券商後台運作機制，這個查詢最佳的執行時段是**交易日的 08:30 到 15:00 之間**。如果你在晚上或半夜呼叫這個函數，主機有可能會拋出錯或回傳過期的昨天資料。
2. **期貨不適用**: 期權交易是保證金與每日洗價結算制度 (Mark to Market)，並無「T+2 日」的概念。如果強行帶入 `--account-kind futures` 可能會引發 API 報錯。
3. **並非即時餘額**: 這個指令只幫你算「這筆股票買賣產生多少交割金」，它**不會**看你現在銀行戶頭裡到底有多少錢。如果要看裡面還有幾塊錢，請搭配 `accounts balance`。
