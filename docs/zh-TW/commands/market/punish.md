# market punish

## 功能定位
查詢並列出當前處於「處置狀態」（處置股）的市場標的，幫助交易者避開流動性受限或委託要求特殊的股票。

## 適用情境
- 策略下單前過濾掉可能受全額交割或分盤交易限制的商品
- 構建選股池時，排除流動性異常或高風險的標的
- 盤中即時監控處置股票名單變更

## 命令語法
```powershell
python scripts/shioaji_cli.py market punish
```

## 參數詳解
此命令目前無需任何參數，預設撈取全市場的處置股。
| 參數 | 必填 | 預設值 | 可選值 | 說明 |
|---|---|---|---|---|
| 無 | | | | |

## 核心回傳資料結構
回傳陣列包含被處置的個股資訊：
- `code`: 股票代號
- `start_date`: 處置開始日期
- `end_date`: 處置結束日期
- `interval`: 分盤撮合時間間隔（分鐘）
- `unit_limit`: 單筆委託張數限制
- `total_limit`: 當日累計委託張數限制
- `description`: 處置詳細說明文字

## 實戰範例
查詢全市場處置股名單：
```powershell
python scripts/shioaji_cli.py market punish
```

## 注意事項與雷區
- **委託限制**：處置股通常伴隨著嚴格的委託限制（如超過一定張數需先圈存），系統不會主動攔截委託，投資人需根據此命令回傳的資訊自行處理風控。
- **資料即時性**：證交所公告可能影響盤前生效狀態，建議於盤前（08:00 後）查詢以獲取最新名單。
