# market snapshots

## 功能定位

這是一個全景市場掃描命令：**抓取商品快照 (Snapshots)**。
與 `kbars` 抓取單一標的長天期歷史紀錄不同，`snapshots` 用於「同時抓取多檔商品在這一瞬間的綜合動態」。它是一次性、非串流的請求，能瞬間返回這些商品今天的開高低收與累計總量。

## 適用情境

- 盤中即時選股：如果你有 50 支口袋名單，你想在 09:30 瞬間知道哪幾支漲幅大於 5%。
- 不想維持 websocket 連線，只想用 Cronjob 每 5 分鐘截一張市場剖面圖。
- 繪製自選股的總覽儀表板。

## 命令語法

```powershell
python scripts/shioaji_cli.py market snapshots --codes <代號1> [<代號2>...] 
```

## 參數詳解

| 參數 | 必填 | 預設值 | 說明 |
|---|---|---|---|
| `--codes` | **是** | 無 | 目標商品代碼清單，**支援輸入多個代碼，以空格分隔**（例如 `--codes 2330 2317 2454`）。 |
| `--market`| 否 | `auto` | 如果你的這批代號全都是股票或全都是期貨，可指定以優化搜尋。 |

---

## 核心回傳資料結構 (`Snapshot` 物件清單)

成功執行後，會回傳一個包含各標的狀態的陣列。重點關注以下欄位：

| 欄位名稱 | 型別 | 說明與實務應用 |
|:-------|:----|:-----------|
| `code` | `str` | 該商品代號。由於回傳陣列不保證按照你輸入的順序，須依賴此欄位核對。 |
| `ts`   | `int` | 此快照資料的產生時間戳（最後一筆市場異動時間）。 |
| `open` / `high` / `low` / `close` | `float` | 今日以來的開盤價、最高、最低、以及目前的最新成交價(或昨收價，若未開盤)。 |
| `change_price` | `float` | 較昨日收盤價的絕對漲跌金額。大於 0 表示漲。 |
| `change_rate`  | `float` | **漲跌幅度 (%)**。程式交易中最常拿來做快篩排序的指標。 |
| `volume`       | `int`   | 最新一筆成交的單筆交易口數/張數。 |
| `total_volume` | `int`   | 今日開盤至當下的**累計總成交量**。可用來篩選市場熱門飆股。 |
| `buy_price` / `sell_price`| `float`| 目前市場上的五檔最優買進價與賣出價。下市價單滑價風險的評估依據。 |

## 實戰與進階範例

### 1. 單一或少量標的快照
一次看台積電 (2330)、鴻海 (2317)、聯發科 (2454) 目前的價位：
```powershell
python scripts/shioaji_cli.py market snapshots --codes 2330 2317 2454 --market stock
```

### 2. 給予大量標的以產出大盤熱度表
結合腳本給入多個重點權值股：
```powershell
python scripts/shioaji_cli.py market snapshots --codes 2330 2317 2454 2881 2891 1301 2002 
```

## 注意事項與雷區

1. **單次數量上限**: 根據券商主機的規定，**一次呼叫 snapshots 的商品數量上限為 500 檔**。如果你試圖把所有上市櫃股票 1700 多檔全部餵給 `--codes`，這道請求將會被主機直接退回/拋出異常。若要掃描全市場，請實作分頁批次呼叫（例如切成 4 個 batch）。
2. **非即時串流**: `snapshots` 不是 websocket。如果你需要 1 毫秒不漏的「逐筆」監視以進行高頻交易，請使用 `quote subscribe`。它只代表你按下 Enter 那零點幾秒鐘的市場橫切面。
3. **未成交或未開盤的空值**: 如果標的今天還沒有任何一筆成交，部分欄位（如 `volume`, `change_price`）可能為 0 或 null，這在處理資料時必須防呆。
