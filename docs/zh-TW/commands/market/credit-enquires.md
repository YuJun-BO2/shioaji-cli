# market credit-enquires

## 功能定位
查詢指定股票清單的信用交易（融資融券）額度與狀態，有助於確認標的是否有足夠的信用額度可供操作。

## 適用情境
- 執行做空前確認是否有融券配額
- 欲使用融資買進前確認融資餘額
- 策略交易系統盤前籌碼或交易限制檢查

## 命令語法
```powershell
python scripts/shioaji_cli.py market credit-enquires --codes <股票代號1> [<股票代號2> ...]
```

## 參數詳解
| 參數 | 必填 | 預設值 | 可選值 | 說明 |
|---|---|---|---|---|
| `--codes` | 是 | 無 | 不限 | 要查詢信用狀態的股票代號列表，可用空格分隔多個代號 |

## 核心回傳資料結構
以 JSON 陣列形式回傳多檔股票的信用交易狀態，每筆項目包含：
- `update_time`: 狀態更新時間
- `system`: 所屬交易系統（例如：TSE）
- `stock_id`: 股票代號
- `margin_unit`: 可用融資張數
- `short_unit`: 可用融券張數

## 實戰範例
查詢台積電 (2330) 與國泰金 (2890) 的信用交易狀態：
```powershell
python scripts/shioaji_cli.py market credit-enquires --codes 2330 2890
```

## 注意事項與雷區
- **查詢上限**：單次查詢數量視實際 API 限制而定，建議分批查詢以免超載。
- **無效代碼**：若傳入無效或不支援信用交易的代碼，回傳結果可能省略該檔或拋出查無合約錯誤。
- **盤中變動**：信用狀態額度在盤中隨時變化，查詢結果僅代表當下快照。
